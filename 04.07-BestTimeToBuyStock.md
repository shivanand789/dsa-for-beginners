# LeetCode 344 Best Time to Buy and Sell Stock 

## Problem

You are given an array `prices` where:

* `prices[i]` represents the stock price on day `i`.
* You may:

  * Buy one stock once.
  * Sell one stock once.
  * You must buy before you sell.

Return the **maximum profit** you can achieve.

### Example

```python
prices = [7, 1, 5, 3, 6, 4]
```

Buy at `1` and sell at `6`.

```text
Profit = 6 - 1 = 5
```

Output:

```python
5
```

---

## Intuition

To maximize profit, we need:

1. The **lowest buying price** seen so far.
2. The **highest profit** achievable if we sell today.

Instead of checking every pair of days (`O(n²)`), we can scan the array once while maintaining:

* `min_price` → minimum stock price seen so far.
* `max_profit` → maximum profit seen so far.

For every price:

* Update the minimum buying price.
* Calculate the profit if we sell today.
* Update the maximum profit.

---

## Algorithm

For each `price` in `prices`:

1. Update minimum price:

```python
min_price = min(min_price, price)
```

2. Calculate current profit:

```python
profit = price - min_price
```

3. Update maximum profit:

```python
max_profit = max(max_profit, profit)
```

---

## Code

```python
class Solution:
    def maxProfit(self, prices):
        min_price = prices[0]
        max_profit = 0

        for price in prices:
            min_price = min(min_price, price)

            profit = price - min_price

            max_profit = max(max_profit, profit)

        return max_profit
```

## Is This Kadane's Algorithm?

Not exactly.

Although it may feel similar because we're continuously updating an optimal answer while scanning the array, this solution is actually a **running minimum (greedy) approach**.

At every step, we ask:

1. What is the minimum price I've seen so far?
2. If I sell today, what profit do I make?
3. Is this the best profit so far?

This greedy observation allows us to solve the problem in a single pass.
